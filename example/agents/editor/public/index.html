What this is even showing

The problem

AXL is peer-to-peer. Your agents address other agents by public key, and those agents talk to peers you've never heard of. Three hops in, something errors and your agent has no idea. No log, no trace, no path back. The deeper the call chain, the darker it gets.

What's running on this page

Five agents on five separate AXL nodes. The Editor (the page you're on) only has two peer keys directly, Researcher and Fact-Checker. Web-Search and Citation-DB sit one hop further out and the Editor has never directly addressed them. They're reached only through their upstream peers. 

Editor ──▶ Researcher   ──▶ Web-Search
   └────▶ Fact-Checker ──▶ Citation-DB   ← rigged to fail
The failure is on purpose. The default prompt feeds Citation-DB a junk source it's hard-coded to reject (demo-fail.example.com). The whole demo is watching that error come back home. Without the sidecar the Editor just knows "fact_check failed" and that's it. With it, the Editor sees the exact peer, the exact hop, and the real error message from two agents away.

Two modes

Sidecar ON · this run

All four downstream peers light up as their spans come home over AXL. Citation-DB's error shows up word-for-word on the leaf node and walks back up the chain to the Editor. Open Jaeger and you'll see the full trace tree, five services, the red span two hops deep with the exception attached.

Sidecar OFF · the dark run

Only the two direct peers light up. The Editor reports a generic "fact_check failed" because nothing about Citation-DB ever made it home. Same workflow, same crash, no trace. This is what debugging on AXL looks like today.

How it works

The sidecar runs next to every agent and pretends to be an OpenTelemetry collector. Each span gets stamped with the originating peer's ID via standard OTel baggage. If the span belongs to this peer the sidecar forwards it to Jaeger. If it belongs to someone else the sidecar ships it home over AXL itself. No central collector, no client library to import, no shared infra.

What to try

  1. Hit Run workflow. Watch the mesh build itself as spans land.
  2. Click Open Jaeger ↗ on any run to see the full trace tree, including spans from peers the Editor never directly talked to.
  3. Expand the red Citation-DB span in Jaeger to see the exception that travelled two hops home.
