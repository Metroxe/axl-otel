AXL OTel — what you're looking at

This is a demo of distributed tracing over an AXL peer-to-peer mesh. Five independent agents — the Editor and four downstream peers — run on separate AXL nodes. The Editor only knows two of them by public key; the other two are reached through those peers. A small sidecar turns AXL into an OpenTelemetry transport so every span comes home to a single Jaeger UI, even from peers the Editor never directly addresses.

The error you see is intentional. The default prompt points the Fact-Checker at a domain the Citation-DB is hard-coded to reject. The whole point of the demo is to watch that failure propagate home over OTel — without the sidecar, the Editor only knows "fact_check failed"; with it, the Editor sees the exact peer, hop, and message that broke.

What to try

  1. Click Run workflow. Watch the mesh build itself as spans arrive.
  2. The directly-called peers (Researcher, Fact-Checker) always light up. The deeper peers (Web-Search, Citation-DB) only appear if their spans came home over AXL.
  3. Click Open Jaeger ↗ on any run to see the full trace tree, including spans from peers the Editor never opened a connection to.

Two modes

  • Sidecar ON (this is what you're running): All four downstream peers light up. The Citation-DB error shows up verbatim on the leaf node and propagates back up the chain.
  • Sidecar OFF (the "broken" demo): Only the two direct peers light up. The Editor reports a generic "fact_check failed" — same workflow, same crash, no path home.

Source & spec: github.com/Metroxe/axl-otel
