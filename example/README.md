# AXL OTel example — 5-agent mesh

A reproducible demo of the [SPEC](../SPEC.md): five agents running on a
local Docker bridge network, each with its own AXL node, Python MCP
router, OTel sidecar, and agent code. The Editor orchestrates a workflow
with Claude Sonnet 4.6; every span from every peer lands in a single
Jaeger instance on the originator's machine.

## What you get

| Agent         | Role                                                                |
| ------------- | ------------------------------------------------------------------- |
| Editor        | Originator. Claude-driven orchestrator. Frontend on `localhost:8080`. |
| Researcher    | MCP server. Calls Web-Search.                                       |
| Web-Search    | Leaf MCP server. Hardcoded fixture corpus.                          |
| Fact-Checker  | MCP server. Calls Citation-DB.                                      |
| Citation-DB   | Leaf MCP server. Hardcoded reputability table.                      |
| Jaeger        | All-in-one image. UI on `localhost:16686`.                          |

Call pattern: `Editor → Researcher → Web-Search` and
`Editor → Fact-Checker → Citation-DB`.

## Prerequisites

- Docker + `docker compose` v2
- `bash`, `openssl`, `xxd` for the key-generation scripts
- An Anthropic API key with access to `claude-sonnet-4-6`

## Quickstart

```bash
# 1. Configure secrets. Docker Compose auto-loads example/.env.
cp example/.env.example example/.env
$EDITOR example/.env      # paste your ANTHROPIC_API_KEY

# 2. Generate ed25519 keypairs + per-agent AXL node configs + peer-id map.
cd example
./scripts/setup.sh

# 3. Build and launch the whole mesh.
docker compose up --build
```

First build pulls `golang:1.25` and `oven/bun:1.3`, clones the AXL repo,
compiles `axl-node`, and installs the Python MCP router. Expect ~5 min
on a cold cache; subsequent builds are near-instant.

When the logs settle, open:

- **Editor UI** — <http://localhost:8080>
- **Jaeger UI** — <http://localhost:16686>

Type a prompt into the Editor, click "Run workflow", and watch the
right-hand panel for the live mesh activity. The final composed answer
appears under the textarea; the full trace (including spans from the
four remote agents, routed over AXL) shows up in Jaeger under the
`editor` service.

## Configuration

All configuration flows through `example/.env`. Compose substitutes
values into `docker-compose.yml` at `up` time — nothing else reads it.

| Variable            | Required | Notes                                        |
| ------------------- | -------- | -------------------------------------------- |
| `ANTHROPIC_API_KEY` | yes      | Used by the Editor to call Claude.           |

Per-agent tuning (model, OTel endpoint, ports, etc.) is hardcoded into
each agent's Dockerfile `ENV` block; override via `environment:` entries
in `docker-compose.yml` if you need to.

## Resetting

```bash
docker compose down -v     # stop containers and clear volumes
rm -rf keys/ configs/      # regenerate keys on next setup
```

`keys/` and `configs/` are gitignored and regenerated by
`scripts/setup.sh`. `.env` is also gitignored — keep it local.

## What's actually happening under the hood

Each container runs four processes via its entrypoint:

1. **`axl-node`** — the AXL Go binary. Yggdrasil bridge + HTTP API on
   `127.0.0.1:9002`.
2. **`mcp_routing.mcp_router`** — the Python MCP router that ships with
   AXL, on `127.0.0.1:9003`. Agent MCP servers register with it at
   startup so inbound `/mcp/{peer}/{service}` traffic can be dispatched
   to the right backend.
3. **`axl-otel` sidecar** — listens on `127.0.0.1:4318` for OTLP HTTP.
   Reads each span's `originator_peer_id` attribute; if it matches this
   node, forwards to the Jaeger container via OTLP. Otherwise ships
   the span to the originator over AXL `/send`. The Editor's sidecar
   additionally runs in `--receive` mode: it polls AXL `/recv` for
   inbound spans from remote peers and writes them into Jaeger.
4. **Agent code** — standard OpenTelemetry SDK, standard MCP JSON-RPC
   over HTTP. The Editor additionally runs an Anthropic-SDK-driven
   tool-use loop on top.

The only thing propagating the originator across hops is plain
OpenTelemetry: the Editor sets a baggage entry
`originator_peer_id = <its own peer id>`, `W3CBaggagePropagator`
injects it into outbound MCP calls' `params._meta`, each downstream
agent extracts it from inbound `params._meta`, and
`BaggageSpanProcessor` stamps it onto every span they emit. No custom
wire protocol and no shared library between agents.
